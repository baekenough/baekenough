Custom Skills

# .claude/skills/observe/SKILL.md --- name: observe description: Record structured memory observations from a primary session --- ## Instructions 1. Analyze the provided primary session data 2. Produce complete, non-truncated XML observations covering: key decisions, tools used, outcomes, and blockers 3. Write exactly ONE progress checkpoint summary per logical milestone 4. Never produce duplicate checkpoints for the same state 5. If rate-limited, stop immediately and report the error — do NOT retry ## Output Format <observation type="{type}" priority="{high|medium|low}"> <content>{observation}</content> <context>{session_context}</context> </observation> <progress_summary milestone="{milestone}"> {summary} </progress_summary>
Headless Mode
Run Claude non-interactively from scripts and CI/CD.
Why for you: Your automated observer/checkpoint system is already trying to run non-interactively but is doing so poorly — spamming 40-200+ retries with no error handling. Switching to headless mode with proper --allowedTools constraints and wrapping it in a script with timeout/retry logic would make your automation robust.
#!/bin/bash # robust-observer.sh - Replace your current automated checkpoint system MAX_RETRIES=3 TIMEOUT=120 for i in $(seq 1 $MAX_RETRIES); do OUTPUT=$(timeout $TIMEOUT claude -p "Analyze this session and create memory observations: $(cat /tmp/primary_session.txt)" \ --allowedTools "Read,Bash,Task" \ --max-turns 10 2>&1) if echo "$OUTPUT" | grep -q "rate limit\|usage limit"; then echo "Rate limited, waiting 60s (attempt $i/$MAX_RETRIES)" sleep 60 continue fi echo "$OUTPUT" exit 0 done echo "ERROR: All retries exhausted" >&2 exit 1
